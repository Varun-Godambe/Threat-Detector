Understanding the Analysis Engine

For Log Files (.log, .txt)

When a log file is uploaded, the software performs a comprehensive incident analysis and presents the findings in a multi-tabbed report:

  • Executive Summary

    A high-level overview of the security incident, identifying the nature of the attack and the primary accounts involved.

  • Anomaly Report

    A detailed list of specific suspicious activities detected by the AI agents, including automated behavior, suspicious simultaneous logins, unusual API endpoint usage, and patterns of session expiration and re-login.

  • Threat Intelligence

    A table of all IP addresses found in the log, checked against a simulated threat database to flag known malicious actors.

  • Attack Chain

    A step-by-step visualization of the attack, from initial access to the final action, making the incident easy to follow.

For Other Files (Scripts, Configs, etc.)

When any other type of file is uploaded, ThreatSight performs a universal static analysis and provides a single, concise report with the following information:

  • File Type Identification

    The AI determines the type of file (e.g., PowerShell Script, JSON, Binary) and checks for signatures of disguised executables.

  • Threat Level Assessment

    The file is assigned a threat level, from "None" to "Critical," based on the findings.

  • Security Summary

    A clear, plain-language explanation of what was found, detailing any potential vulnerabilities, dangerous commands, or insecure settings.
