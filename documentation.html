About & Documentation

1. Introduction

ThreatSight is a static analysis tool designed for developers, security analysts, and hobbyists. Its core mission is to provide a quick, accessible, and private way to scan digital assets for common security risks. Unlike other tools, ThreatSight performs all analysis directly in your browser, ensuring your files, logs, and data never leave your machine.

2. Features Deep Dive

ThreatSight is equipped with specialized analyzers for different types of assets.

2.1 File Analysis

The file analyzer inspects the content of scripts and configuration files for known anti-patterns and vulnerabilities.

  • PowerShell (.ps1): Detects suspicious cmdlets (e.g., Invoke-Expression, IEX), Base64 encoded commands, and remote download patterns that are common in malicious scripts.
  • Python (.py): Scans for the use of insecure modules (e.g., pickle, shelve), potential command injection points (e.g., os.system), and hardcoded secrets like API keys.
  • Shell Scripts (.sh): Looks for risky commands (e.g., rm -rf), insecure use of curl or wget, and variables being executed directly.
  • JSON (.json): Checks for keys that might contain sensitive data (e.g., "password", "apiKey") and common misconfigurations.

2.2 Log Analysis

The log analyzer uses pattern matching to identify suspicious activity within raw log data. It can detect patterns indicative of brute-force attempts, multiple failed logins from a single IP, and unusual user-agent strings.

2.3 URL Scanning

This feature performs a basic check of a URL against a hardcoded list of known malicious domains. It serves as a preliminary check for phishing or malware-hosting sites. (Note: This is a demo feature and not a comprehensive blacklist).

3. Privacy by Design

Your data stays yours. We cannot stress this enough. ThreatSight is built entirely with client-side JavaScript. When you upload a file or paste text, the analysis happens locally on your computer. No files, no text, and no metadata are ever transmitted to a server. This makes it safe to analyze sensitive or proprietary code without privacy concerns.

4. Frequently Asked Questions (FAQ)

Is ThreatSight a replacement for an antivirus or a full-fledged SAST tool?
No. ThreatSight is a lightweight, first-pass analysis tool. It is meant to supplement, not replace, professional security software. It's great for quick checks and educational purposes but does not provide the comprehensive protection of an antivirus or a commercial Static Application Security Testing (SAST) solution.
How can I contribute?
This is a personal project, but feedback is always welcome! If you have ideas for new detection rules or features, feel free to reach out or fork the project on GitHub.
