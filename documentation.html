Documentation

ThreatSight is a static analysis tool designed for developers, security analysts, and hobbyists. Its core mission is to provide a quick, accessible, and private way to scan digital assets for common security risks.

The Analysis Agent

Unlike simple text matching, ThreatSight uses a client-side "agent" that performs more intelligent, stateful analysis. This means it doesn't just look for single bad lines, but rather for suspicious patterns and correlations across the entire file or log.

  • Stateful Tracking: The agent remembers information as it scans. For example, it tracks failed login attempts per IP address to accurately identify brute-force attacks.
  • Pattern Recognition: It's designed to recognize common attack chains, such as a vulnerability scan followed by an exploit attempt from the same source.
  • Risk Scoring: Findings are assigned a severity level (Info, Warning, Critical) based on the confidence and potential impact of the detected activity.

Features Deep Dive

The agent is equipped with specialized analyzers for different types of assets.

File Analysis

Inspects the content of scripts for known anti-patterns and vulnerabilities.

  • PowerShell (.ps1): Detects suspicious cmdlets (e.g., Invoke-Expression, IEX), Base64 encoded commands, and remote download patterns that are common in malicious scripts.
  • Python (.py): Scans for the use of insecure modules (e.g., pickle, shelve), potential command injection points (e.g., os.system), and hardcoded secrets like API keys.
  • Shell Scripts (.sh): Looks for dangerous commands (e.g., rm -rf), insecure use of cURL or Wget piped to a shell, and other common anti-patterns.
  • JSON (.json): Checks for keys that might contain sensitive data (e.g., "password", "apiKey") and common misconfigurations.

Log Analysis

Parses common log formats to extract structured data (IP, timestamp, status code, etc.) before analysis. This allows for much more accurate detection of:

  • Brute-Force Attacks: Detects a high number of failed logins (e.g., HTTP 401/403) from a single IP in a short time.
  • Vulnerability Scanning: Identifies IPs that are rapidly requesting non-existent pages, especially known administrative or sensitive paths (e.g., /wp-admin, /.env, /sqladmin).
  • Anomalous User Agents: Flags requests from suspicious or known malicious user-agents (e.g., `sqlmap`, `nmap`, `masscan`).
  • Potential Attack Chains: Correlates scanning behavior with subsequent successful logins or exploit attempts from the same IP address.

Privacy by Design

Your data stays yours. We cannot stress this enough. ThreatSight is built entirely with client-side JavaScript. When you upload a file or paste text, the analysis happens locally on your computer. No files, no text, and no metadata are ever transmitted to a server. This makes it safe to analyze sensitive or proprietary code without privacy concerns.

Frequently Asked Questions (FAQ)

Is ThreatSight a replacement for an antivirus or a full-fledged SAST tool?
No. ThreatSight is a lightweight, first-pass analysis tool. It is meant to supplement, not replace, professional security software. It's great for quick checks and educational purposes but does not provide the comprehensive protection of an antivirus or a commercial Static Application Security Testing (SAST) solution.
How can I contribute?
This is a personal project, but feedback is always welcome! If you have ideas for new detection rules or features, feel free to reach out to me or fork the project on GitHub.
